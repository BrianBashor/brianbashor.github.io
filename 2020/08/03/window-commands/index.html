Find process related to network traffic

I recently noticed some weird keep-alive packets coming from an unknown IP address.

The following commands will correlate network traffic to a given system process.

Get-Process -Id (Get-NetTCPConnection -LocalPort port_#).OwningProcess
netstat -a -b

Turns out to be Razer. Maybe I should have know =/
This can also be done with windows sysinternals
